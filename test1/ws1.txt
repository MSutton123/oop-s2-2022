1. What is the purpose of the log message you supply to your git commit?
The supplied message updates the code to an archive of the code.

2. What do you think makes a good commit message?
A good commit message is simple and easy to read, whilst using lowercase letters.

3. When and how often should I be committing my work?
Work should be committed after each section of code, when testing for bugs and before publication of code.

4. True or False, you should commit all files (including binaries, images and log files) to Git?
True, all work should be committed to git
